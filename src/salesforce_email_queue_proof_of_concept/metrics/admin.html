System reference

• Analytics table: salesforce_email_queue_events (local MySQL; created on startup).
• Ask-your-data tables: salesforce_email_queue_ask_log, salesforce_email_queue_ask_corrections.
• Test mode: open any page with ?metrics_test=1 → events flagged is_test=1, purgeable under Maintenance. Admin/Metrics/App links here carry it.
• No member PII is stored: counts/enums + operator staff username + queue name only.
• Admin pages reuse the app session with role admin (no second login).
• Sensitive data lives OUTSIDE the repo: auth.json, corrections.json, queue_access.json.
• Env: SF_PROD_*, OPENAI_API_KEY/ANTHROPIC_API_KEY, SF_EMAIL_QUEUE_ADMIN_*/SF_EMAIL_QUEUE_*, LOCAL_MYSQL_*.
