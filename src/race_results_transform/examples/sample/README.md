# Sample fixtures — SYNTHETIC, NOT REAL ATHLETES

Everything in this folder is **fabricated test data** (fake names, `@example.com` emails,
made-up member numbers). It is safe to commit and is used by `tests/sample.test.js` so the test
suite has a regression fixture that works on any clone / CI — independent of the real, machine-
local data directory (`usat/data/...`, which is never committed).

- `sample_race_results_FAKE.csv` — hand-authored; covers blank/non-numeric members, mixed gender
  spellings + a typo, ISO/`mm-dd-yy` dates, full/abbrev/foreign states, division-name vs age-band
  categories, a `DNS`, and split/off-template columns that get dropped.
- `sample_race_results_FAKE.xlsx` — generated by `build_sample.js` (run `node build_sample.js`);
  uses native Excel date/time cells so the exceljs read path + "time renders as time" are exercised.
- `expected/*.json` — committed golden snapshots the test compares against.

Regenerate the xlsx after editing the data: `node examples/sample/build_sample.js`.
Real conversions still use `usat/data/...` (via `determineOSPath`); this folder is for tests/demos only.

`sample_fullname_FAKE.csv` has a single **Name** column (no First/Last) to exercise the
full-name split into First Name + Last Name. All data here is fake.
