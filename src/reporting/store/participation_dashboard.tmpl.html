Region flow matrix

Read a row = where that region's athletes race (total "By its athletes"); a column = who races in that region ("Hosted here" bottom row). Diagonal = racing within the same region (incl. their own state). Net ± = Hosted − By-its-athletes: green = net importer (gain), red = net exporter (loss). Reference & definitions

What the data means, how each number is calculated, and what to watch for.

Reading the dashboard

Every control (Year, Month, Map style, Metric, and the Advanced menu) updates the whole page at once. Ctrl/Cmd-click in Year or Month to combine periods. Click a state or region on the map to filter the Events table. Sort any table by clicking its header; every table exports to CSV.

Map styles

  • Choropleth — States shaded by the selected metric.
  • Pins — Event locations plotted by ZIP code.
  • Both — Choropleth shading plus event pins.
  • YoY — Year-over-year change by state (percent or absolute), with numbered badges on the top movers.
  • Flows — 3D arcs showing athlete travel from home state to event state; the base map shades net importers (red) vs exporters (blue).

Core metrics & how they are calculated

  • Participants — Count of participation records (event starts) in the period. One athlete racing 3 times counts as 3.
  • Unique participants — Distinct athletes (deduplicated).
  • % unique — Unique participants ÷ Participants.
  • Races / participant — Participants ÷ Unique participants.
  • Home (in-state) — Participations where the athlete's home state = the event state.
  • Away (cross-state travel) — Participations where home state ≠ event state.
  • Home % — Home ÷ (Home + Away).
  • New / Repeat — First-time vs returning athletes in the period.
  • IRONMAN share — IRONMAN participations ÷ all participations.
  • Female %, age bands — Share of participations in that group.

The matrices (travel)

  • Cell (row = home, col = event) — Participations by that home state/region's athletes racing in that event state/region.
  • Diagonal — State matrix: raced in their own state (in-state). Region matrix: raced within their own region (includes traveling to another state in the same region).
  • By its athletes (row total) — All participations by that state/region's athletes (in-state + traveled).
  • Hosted here (column total) — All participations held there (locals + visitors).
  • Traveled out — By its athletes − diagonal.
  • Traveled in — Hosted here − diagonal.
  • Net ± — Hosted here − By its athletes (= Traveled in − Traveled out). Positive = net importer / destination; negative = net exporter / feeder.
  • Grand total travel (dark cell) — Sum of all off-diagonal cells = total participations − in-state races.
  • Region matrix, two measures — "Out/Into region" = crossed a region line (reconciles with the diagonal); "Out/Into state" = raced in any other state (total cross-state travel, incl. trips within the same region).

Cross-state travel — why there are two totals

Both measure the same idea (racing outside the home state) from different sources:

  • Traveled-away KPI (Summary totals) — Counted per participation record; includes away trips whose destination is outside the 50-state grid (DC, territories). The fullest total.
  • Matrix grand-total cell — Sums only flows between the 50 mapped states, so it is slightly lower by design.

Example (2025): Participants 292,675 · Home 178,019 · Away 84,706 · matrix between-state travel 78,636 · records with no usable home state ~29,950. Home + Away can fall short of Participants because of those unmatched records.

Year-over-year

  • % change — (to − from) ÷ from.
  • Absolute — to − from.
  • From zero — A state with 0 in the from-year has no valid percentage (division by zero); it is shown as "0 → N" and excluded from the Top-N gainers ranking.
  • Top movers — Numbered gold badges mark the ranked gainers/decliners.

Data, freshness & caveats

  • Source — race_director_update_skeleton (State/Region Metrics + State × Event origin-destination pairs). Boundaries via Plotly; region outlines from us-atlas.
  • Last updated — Shown in the header (source table timestamp).
  • 2026 — Year-to-date (partial) — not comparable to full years.
  • Multi-period selections — Participations, gender, age, home/away, IRONMAN and new/repeat are exact. Distinct counts (Unique participants, Events, Races) are summed per period, so an athlete or event spanning two periods can be counted more than once. A single year with all/one month is exact.
